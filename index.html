DECEMBER 3rd 2017 #2

A Simple Gallery



This is a very simple static gallery that uses HTML and CSS to display named images drawn from a folder hosted by Github.

The next stage is to write a Ruby script to bring random images from a folder hosted on my cloud accounts. I want to use other sources for the images because Github only offers 1GB of space for the free web hosting service - and I have 100GB of storage on services like Box and Mega which I want to use to store media (images, movies, audio, documents etc) for use in this project.

This is the CSS code for the gallery (use div class equals gallery in the HTML):

div.gallery {
padding: 10px;
align: middle;
width: auto;
height: auto;}

div.gallery:hover {
border: 0px;
}

div.gallery img {
width: 100px;
height: 100px;
}

If you prefer a reactive screen on 'hover' then change the border to something bigger than zero pixels.

DECEMBER 2nd 2017

I recently retired. I am new to programming. I am interested in learning IT skills as a hobby and to keep my mind active. Use it or lose it!

I have Windows, Apple, Android (open source and propriatory), Firefox, Sailfish, and Ubuntu portable devices.

I buy second-hand items and play with them. I like taking the hardware apart and putting it together again - I have always done that with technology, ever since I was a little girl. I think I should have been an engineer. Hands-on is how I learn best.

I also like flashing different open source operating systems onto my devices (yes, I have bricked a few) but apart from this I only use them to consume content.

I prefer to do creative projects that involve a lot of screen-time (e.g. editing photographs or video footage) and/or typing (e.g. writing a report) sitting in a proper office chair, at a desk. My laptop runs Linux Mint so my long-term ambition it to learn to programme in Linux Mint. My short-term aim is to find resources to teach myself programming that are open source, free, and can be installed on my Linux Mint laptop.

I looked online for advice about which programming language I should start with. I decided to try teaching myself to use Ruby and work up to using other languages.

Today, I made my first attempt to use GitHub to build (using HTML and CSS) and host a simple website. Once I have a basic website, I will try to use Ruby to add features to it.

Github provides a tutorial to get newbies started and it is quite good but it leaves out many essential things. For example, it shows you how to create files but not how to create folders. This omission shows the beginner's guide was not written by a beginner but by someone who understands Github inside-out and thus has a blind-spot about the basics, assuming knowledge instead of imparting it.

If you are wondering, you create folders on Github by typing them into the same box used to create files - for example, to create a folder called colours with a text file inside called red.txt all you do is type /colours/red.txt and Github will create the folder and file for you. It cannot create empty folders so create a dummy file and save it to make a folder and then delete the dummy file when no longer required.



I tried the various WYSIWYG website builders compatible with Linux Mint like Jekyll and could not make any progress with them.

I tried Github Pages and drew another blank.

I then downloaded and installed the Atom editor for Linux distros and just typed the codes manually and then uploaded them to my Github repository. And this website is the result.

I suspect that the terminal-driven short-cuts like Jekyll are only useful if you are already quite skilled. They are promoted for newbies but I find that actually as a total newbie I prefer the simple keyboard bashing process. Maybe the physical effort of typing code helps me understand the logic and commit the process to memory?
